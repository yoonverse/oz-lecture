CSS Float와 Clear

float로 생기는 문제와 clear로 해결하는 방법을 비교합니다.

1. Float 기본 동작

float을 적용하면 요소가 왼쪽 또는 오른쪽으로 떠오르고, 주변 텍스트가 감싸는 형태로 배치됩니다.

float: left
float: left

텍스트가 float된 박스의 오른쪽을 감싸며 흘러갑니다. float는 원래 이미지 주변에 텍스트를 배치하기 위해 만들어진 속성입니다.

float: right
float: right

텍스트가 float된 박스의 왼쪽을 감싸며 흘러갑니다. float: right는 요소를 오른쪽으로 밀어서 배치합니다.

2. Float의 문제 - 다음 요소가 겹침

float된 요소 다음에 오는 블록 요소는 float를 무시하고 겹쳐서 배치됩니다. clear를 사용하면 이 문제를 해결할 수 있습니다.

clear 없음 - 다음 요소가 겹침
float: left
다음 블록 요소 (clear 없음) - float 박스 뒤로 들어가 겹칩니다!
/* 다음 요소에 clear 없음 */ .next { /* clear 속성 없음 */ }
clear: both 적용 - 겹침 해결
float: left
다음 블록 요소 (clear: both) - float 박스 아래로 내려갑니다!
/* 다음 요소에 clear 적용 */ .next { clear: both; }

3. Float의 문제 - 부모 요소의 높이 붕괴

자식 요소가 모두 float되면 부모는 자식의 높이를 인식하지 못해 높이가 0이 됩니다. 점선 테두리가 부모 .container입니다. clearfix 패턴으로 해결합니다.

clearfix 없음 - 부모 높이 붕괴
Box 1
Box 2
Box 3
부모 밖의 요소 - 부모 높이가 0이라 바로 위에 붙습니다!
/* 부모에 clearfix 없음 */ .container { /* 자식이 모두 float → 높이 0 */ }
clearfix 적용 - 부모가 높이를 인식
Box 1
Box 2
Box 3
부모 밖의 요소 - 부모가 자식 높이를 정상적으로 감쌉니다!
/* clearfix 패턴 */ .container::after { content: ""; display: block; clear: both; }

4. 실전 활용 - Float으로 가로 배치

float를 활용하면 블록 요소를 가로로 배치할 수 있습니다. 단, 반드시 clearfix를 함께 사용해야 레이아웃이 깨지지 않습니다.

clearfix 없이 가로 배치
사이드바
메인 콘텐츠
푸터 - 컨테이너가 붕괴되어 레이아웃이 깨집니다!
clearfix와 함께 가로 배치
사이드바
메인 콘텐츠
푸터 - clearfix 덕분에 정상 배치됩니다!
